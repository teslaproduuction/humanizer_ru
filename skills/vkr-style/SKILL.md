# Стиль академической технической документации (ВКР)

Данный skill предназначен для формирования текстов в академическом стиле, соответствующем требованиям к выпускным квалификационным работам, научным публикациям и технической документации в сфере высшего образования.

## Основные принципы

### 1. Формальность и профессионализм

**Требования:**
- Использование научного стиля изложения
- Отсутствие эмоциональной окраски и разговорных конструкций
- Нейтральный тон повествования
- Объективность в представлении информации

**Примеры:**

| Неформальный стиль | Академический стиль |
|-------------------|---------------------|
| Мы решили использовать этот подход | В рамках исследования был применён данный подход |
| Получилось довольно интересно | Результаты демонстрируют значимые характеристики |
| Это работает лучше | Данный метод показывает более высокую эффективность |

### 2. Структурированность изложения

**Обязательные элементы структуры:**

1. **Чёткая логическая последовательность**
   - Введение → Основная часть → Заключение
   - Переход от общего к частному
   - Последовательное раскрытие темы

2. **Использование связующих элементов**
   - "В рамках данного исследования..."
   - "На основании проведённого анализа..."
   - "Таким образом..."
   - "Следовательно..."
   - "В результате..."

3. **Нумерация и иерархия**
   - Многоуровневая нумерация разделов (1, 1.1, 1.1.1)
   - Структурированные списки
   - Чёткое выделение подразделов

### 3. Терминологическая точность

**Требования к терминологии:**

- Использование устоявшихся научных терминов
- Консистентность терминологии в пределах документа
- Определение специальных терминов при первом упоминании
- Избегание синонимичных замен ключевых терминов

**Пример:**
```
При первом упоминании: "Искусственная нейронная сеть (ИНС) представляет собой..."
В дальнейшем: использование либо полного термина "искусственная нейронная сеть", 
либо аббревиатуры "ИНС", но не произвольных синонимов.
```

### 4. Академические обороты речи

**Рекомендуемые конструкции:**

**Описание методологии:**
- "Методология исследования базируется на..."
- "В ходе работы применялись следующие методы..."
- "Исследование проводилось в несколько этапов..."
- "Для решения поставленных задач использовались..."

**Представление результатов:**
- "Результаты исследования демонстрируют..."
- "Полученные данные свидетельствуют о..."
- "Анализ показал, что..."
- "Экспериментальная проверка подтвердила..."

**Формулирование выводов:**
- "На основании проведённого исследования можно заключить..."
- "Таким образом, установлено, что..."
- "Полученные результаты позволяют утверждать..."
- "В результате выполненной работы разработано/предложено/создано..."

### 5. Описание технических решений

**Структура описания технического решения:**

1. **Обоснование необходимости**
   - "Для решения данной задачи требуется..."
   - "С целью обеспечения... необходимо..."

2. **Описание подхода**
   - "Предлагается использовать следующий подход..."
   - "Реализация осуществляется посредством..."

3. **Технические характеристики**
   - Точные параметры (без приблизительных оценок)
   - Численные показатели с единицами измерения
   - Ссылки на стандарты и спецификации

4. **Обоснование выбора**
   - "Выбор данного решения обусловлен..."
   - "Преимущества данного подхода заключаются в..."

**Пример:**
```
Для реализации системы управления была выбрана микроконтроллерная платформа 
ESP32-WROOM-32 с тактовой частотой 240 МГц и объёмом оперативной памяти 520 КБ. 
Выбор данной платформы обусловлен наличием встроенных модулей Wi-Fi и Bluetooth, 
что обеспечивает возможность беспроводной передачи данных без использования 
дополнительных компонентов.
```

### 6. Оформление ссылок и цитирования

**Правила оформления:**

- Ссылки на источники в квадратных скобках: [1], [2, с. 45]
- При цитировании: точное воспроизведение текста в кавычках
- Указание страниц при цитировании
- Оформление списка литературы по ГОСТ

**Примеры:**
```
Согласно [1], данный подход демонстрирует высокую эффективность.
Как отмечает автор работы [2, с. 34]: "Применение нейронных сетей..."
```

### 7. Использование специализированной лексики

**Типичные обороты для ВКР:**

**Постановка задачи:**
- "Целью данной работы является..."
- "Задачи исследования включают..."
- "Объектом исследования выступает..."
- "Предметом исследования является..."

**Описание актуальности:**
- "Актуальность темы обусловлена..."
- "Практическая значимость работы заключается в..."
- "Научная новизна определяется..."

**Описание процесса:**
- "Был проведён анализ..."
- "Осуществлялась разработка..."
- "Выполнялось тестирование..."
- "Производилась оптимизация..."

### 8. Избегаемые конструкции

**НЕ использовать:**

❌ Личные местоимения от первого лица единственного числа ("я", "мне", "мой")
❌ Разговорные выражения и жаргонизмы
❌ Эмоционально окрашенную лексику
❌ Неопределённые формулировки ("примерно", "около", "приблизительно" - без обоснования)
❌ Вопросительные предложения в основном тексте (кроме введения при постановке проблемы)
❌ Восклицательные предложения
❌ Сокращения без предварительной расшифровки

**Вместо этого использовать:**

✓ Безличные конструкции ("был проведён анализ", "выполнена разработка")
✓ Формы множественного числа ("мы", "нами" - допустимо)
✓ Точные численные значения с указанием погрешности
✓ Утвердительные конструкции
✓ Полные формы при первом упоминании с аббревиатурой в скобках

### 9. Оформление результатов исследования

**Представление данных:**

1. **Таблицы:**
   - Нумерация: "Таблица 1 – Название таблицы"
   - Ссылка в тексте: "Результаты представлены в таблице 1"
   - Единицы измерения в заголовках столбцов

2. **Рисунки:**
   - Нумерация: "Рисунок 1 – Название рисунка"
   - Обязательная ссылка в тексте перед рисунком
   - Подпись под рисунком

3. **Формулы:**
   - Нумерация справа: (1), (2), (3)
   - Расшифровка обозначений после формулы
   - Ссылки в тексте: "согласно формуле (1)"

**Пример оформления:**

```
Эффективность системы рассчитывается по формуле:

η = (Pout / Pin) × 100%,                    (1)

где η – коэффициент полезного действия, %;
    Pout – выходная мощность, Вт;
    Pin – входная мощность, Вт.

Полученные значения представлены в таблице 1.

Таблица 1 – Результаты измерений эффективности системы

| № п/п | Pin, Вт | Pout, Вт | η, % |
|-------|---------|----------|------|
| 1     | 100     | 87       | 87   |
| 2     | 150     | 128      | 85,3 |
```

### 10. Специфика технической документации

**Для описания программного обеспечения:**

- Указание версий используемых библиотек и фреймворков
- Описание архитектуры с использованием UML-диаграмм
- Детализация алгоритмов работы
- Спецификация требований (функциональных и нефункциональных)

**Для описания аппаратных решений:**

- Точные технические характеристики компонентов
- Схемы подключения и взаимодействия
- Обоснование выбора элементной базы
- Расчёты параметров (с формулами и численными результатами)

**Пример технического описания:**

```
Система построена на базе микроконтроллера ATmega328P с архитектурой AVR. 
Тактовая частота процессора составляет 16 МГц, объём flash-памяти – 32 КБ, 
SRAM – 2 КБ, EEPROM – 1 КБ. Для программирования использовалась интегрированная 
среда разработки Arduino IDE версии 1.8.19 с компилятором avr-gcc версии 7.3.0.

Связь с периферийными устройствами реализована посредством интерфейсов:
- UART (скорость передачи данных 9600 бод);
- I2C (частота шины 100 кГц);
- SPI (частота шины 4 МГц).
```

### 11. Стилистические требования к различным разделам ВКР

**Введение:**
- Обоснование актуальности темы
- Формулирование цели и задач
- Определение объекта и предмета исследования
- Описание методов исследования
- Краткая характеристика структуры работы

**Аналитический раздел:**
- Обзор существующих решений
- Сравнительный анализ подходов
- Выявление преимуществ и недостатков
- Формулирование требований к разрабатываемому решению

**Проектный/практический раздел:**
- Описание разработанного решения
- Обоснование проектных решений
- Детальное описание реализации
- Представление результатов тестирования

**Заключение:**
- Краткое изложение выполненной работы
- Формулирование полученных результатов
- Оценка степени достижения поставленных целей
- Направления дальнейших исследований

- [ ] Правильная нумерация разделов
- [ ] Использование безличных конструкций
- [ ] Отсутствие разговорных выражений

## Примеры трансформации текста

### Пример 1: Неформальный → Академический

**Неформальный:**
> Я решил использовать ESP32, потому что он дешёвый и у него есть Wi-Fi. Сделал программу на Python, которая собирает данные с датчиков. Результаты получились хорошие.

**Академический:**
> В качестве аппаратной платформы была выбрана микроконтроллерная система ESP32-WROOM-32, что обусловлено наличием встроенного модуля Wi-Fi IEEE 802.11 b/g/n и оптимальным соотношением технических характеристик к стоимости. Программное обеспечение для сбора данных с измерительных датчиков реализовано на языке программирования Python версии 3.9 с использованием библиотеки micropython. Экспериментальная проверка разработанной системы продемонстрировала соответствие заданным техническим требованиям.

### Пример 2: Описание алгоритма

**Неформальный:**
> Сначала программа читает данные, потом их обрабатывает и выводит результат на экран.

**Академический:**
> Алгоритм функционирования программного модуля включает следующие этапы:
> 1. Считывание исходных данных из источника информации посредством интерфейса UART.
> 2. Обработка полученных данных с применением алгоритма цифровой фильтрации (фильтр Калмана).
> 3. Вывод результатов обработки на дисплейный модуль через интерфейс SPI.
> Временная диаграмма выполнения алгоритма представлена на рисунке 2.

### Пример 3: Описание результатов эксперимента

**Неформальный:**
> Протестировал систему - всё работает нормально. Точность около 95%.

**Академический:**
> В ходе экспериментальной проверки разработанной системы было проведено 100 измерений контролируемого параметра. Статистический анализ полученных результатов показал, что средняя абсолютная погрешность измерений составляет 4,7% при доверительной вероятности 0,95. Полученные показатели точности соответствуют требованиям, установленным техническим заданием.

## Заключение

Данный skill обеспечивает формирование текстов, соответствующих академическим стандартам оформления выпускных квалификационных работ, технической документации и научных публикаций. Применение изложенных принципов позволяет создавать тексты с требуемым уровнем формальности, структурированности и терминологической точности.
